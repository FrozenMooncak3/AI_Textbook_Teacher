## Phase 1：MVP——单模块学习闭环

**结束标志**：在本地电脑 localhost:3000 上，能用真实文本跑完完整的单模块学习流程，不需要手动干预。

### 第1步：数据库
- 建表：books / modules / questions / user_responses / mistakes / review_tasks
- 数据库连接和初始化逻辑（app 启动时自动建表）

### 第2步：文件上传
- 后端 API：接收 PDF 文件，服务端提取文本，存入数据库（使用 `pdf-parse` 库）
- 前端页面：上传框 + 填写书名 + 提交

### 第3步：模块地图生成
- 后端：把文本发给 Claude，按 learning_flow.md 规则拆解模块、提取 KP
- 前端：展示模块地图（标题 / KP 数量 / 预计题数 / 核心技能）

### 第4步：读前指引
- 进入模块后，Claude 生成任务锚（能做什么、重点、容易混淆）
- 提供"跳转原文"按钮（在 app 内展示原文文本）
- 用户点"我读完了"进入 Q&A

### 第5步：Q&A
- Claude 根据模块 KP 生成题目
- 一次一题，下方有回答框，点"下一题"继续
- 全部答完后，AI 逐题给出评分 + 解析 + 纠偏

### 第6步：模块测试
- 进入测试前显示软性提醒（建议明天再做）
- 独立作答，提交后 AI 评分
- 通过（≥80%）：模块标记完成

### 第7步：错题诊断与记录
- 未通过：AI 诊断每道错题的错误类型，给出补救建议
- 通过：错题存入 mistakes 表，复习任务写入 review_tasks 表

## Phase 2：完整一本书

**结束标志**：能学完一本书的所有模块，复习任务会按时出现。

- 书籍首页：所有模块进度一览（未开始 / 进行中 / 已通过）
- 多模块进度管理
- 复习提醒系统（3→7→15→30→60 天到期提示）
- 复习任务页面（展示到期任务，完成后更新状态）

---

## Phase 3：上线

**结束标志**：有一个公开网址，任何设备都能打开使用。

- 数据库从 SQLite 迁移到 Supabase（Vercel 不支持本地文件系统）
- 部署到 Vercel
- 环境变量配置
- 基础错误处理和体验打磨

---

## 明确不做（任何阶段）

- 多用户 / 登录系统
- 社区、分享、协作功能
- 多种学习模式切换
- 复杂个性化推荐
