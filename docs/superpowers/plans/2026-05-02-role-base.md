# 角色系统骨架（决策 1+2）实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use `task-execution` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal**: 给 `users` 表加 `role` 列（`'user' | 'admin'`），admin 自动 bypass 上传三层拦截（quota / 1h limit / monthly budget 检查），user_id=1 设为 admin，dogfooding 再也不被卡。

**Architecture**: 三步 — (1) schema migration 加 `users.role` enum 字段；(2) 新建 `src/lib/entitlements.ts` 集中权限判断函数 `canBypassUploadLimits`；(3) 在 `presign` + `confirm` 路由用 `if (!canBypassUploadLimits(user)) { ...拦截逻辑 }` 守卫包住现有 quota/budget 检查。**不改** `cost-meter-service.ts` —— admin 自测的累计成本远低于月预算（DeepSeek 单本 0.5-1 元 × 月 5-10 本 = ~10 元），月预算 500 元的拦截阈值用 admin bypass 即可绕开，无需专门跳过 `monthly_cost_meter` 写入。

**Tech Stack**: PostgreSQL `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` 幂等 migration / Next.js 15 App Router server-side guard / TypeScript 联合类型 / `node:test` + ESM `data:` URL stub 单元测试 / 现有 `requireUser` cookie session 复用

**关联**: [spec](../specs/2026-05-02-role-admin-staging-design.md) 决策 1+2 / [WIP](../specs/2026-05-02-role-admin-staging-brainstorm-state.md)

**触发**: 5/2 临时 SQL `update users set book_quota_remaining = 99 where id = 1` 解锁开发自测，必须改成正式机制。

- 函数名 `incrementCost` → `recordCost`
- 「bypass 写入但保留 cost_log 写入」→「不改 recordCost，admin 月预算累计照常但 bypass 在 isBudgetExceeded 检查侧就够了」

-- Role system (2026-05-02): admin bypass for D7 upload limits
ALTER TABLE users ADD COLUMN IF NOT EXISTS role TEXT NOT NULL DEFAULT 'user';

DO $$
BEGIN
  IF NOT EXISTS (
    SELECT 1 FROM pg_constraint c
    JOIN pg_class t ON t.oid = c.conrelid
    WHERE t.relname = 'users' AND c.conname = 'users_role_check'
  ) THEN
    ALTER TABLE users ADD CONSTRAINT users_role_check
      CHECK (role IN ('user', 'admin'));
  END IF;
END $$;

  return queryOne<User>(
    `
      SELECT
        u.id,
        u.email,
        u.display_name,
        u.role,
        u.book_quota_remaining,
        u.book_quota_remaining + COALESCE(
          (SELECT COUNT(*)::int FROM book_uploads_log WHERE user_id = u.id),
          0

}
