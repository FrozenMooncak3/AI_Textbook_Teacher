## T1 · ocr-auth 加 IdTokenClient 缓存（Codex）

**Files:**
- Modify: `src/lib/ocr-auth.ts:1-40`（当前全文 40 行）

**目标：** `cachedAuth: GoogleAuth` 单例已有，但 `buildOcrHeaders` 每次都跑 `auth.getIdTokenClient(audience)`——ID token 每次都重新走 OAuth 换证。加一个 `Map<audience, IdTokenClient>` 永久缓存。`IdTokenClient` 内部自动管 token TTL，client 永久缓存安全。

**大白话：** 网站每次跟 OCR 服务说话之前都要"去 Google 换一张通行证"。原代码里换完就丢了，下次又去重换。这一步 hang 的嫌疑最大。现在加个"通行证存根"盒子——每个 audience（OCR 服务网址）第一次换完就塞进盒子，下次直接取。

- [ ] **Step 1：读 `src/lib/ocr-auth.ts` 全文（40 行）确认当前结构**

Run: `cat src/lib/ocr-auth.ts`
Expected: 看到 `cachedAuth: GoogleAuth | null` + `getAuth()` + `buildOcrHeaders()`，和本 plan 的改动起点一致。

- [ ] **Step 2：加 `IdTokenClient` import + `clientCache` 模块级 Map**

在 `import { GoogleAuth } from 'google-auth-library'` 下加一行：

```ts
import { GoogleAuth, type IdTokenClient } from 'google-auth-library'
```

在 `let cachedAuth: GoogleAuth | null = null` 下加：

```ts
const clientCache = new Map<string, IdTokenClient>()
```

- [ ] **Step 3：改 `buildOcrHeaders` 的 IAM 分支——先查缓存再建**

把原来的：

```ts
if (requireIamAuth) {
  const auth = getAuth()
  const audience = new URL(targetUrl).origin
  const client = await auth.getIdTokenClient(audience)
  const idHeaders = await client.getRequestHeaders()
  // ...
}
```

改成：

```ts
if (requireIamAuth) {
  const audience = new URL(targetUrl).origin
  let client = clientCache.get(audience)
  if (!client) {
    const auth = getAuth()
    client = await auth.getIdTokenClient(audience)
    clientCache.set(audience, client)
  }
  const idHeaders = await client.getRequestHeaders()
  // ...
}
```

注意：后续 `idHeaders.Authorization || idHeaders.authorization` 和 `headers.Authorization = authHeader` 逻辑保持不变。

## T2 · upload-flow 加 fetch timeout + 1 次重试（Codex）

**Files:**
- Modify: `src/lib/upload-flow.ts:39-156`

**目标：** `runClassifyAndExtract` 里 3 处 `fetch` 当前均无 timeout，hang 走到 Vercel 函数 maxDuration（Fluid 300s）。每处加 `AbortSignal.timeout(30_000)` + 复用项目自有 `retryWithBackoff`（`maxAttempts: 2, baseMs: 1000` = 原始 1 次 + 重试 1 次，退避 1s）。`classifyError` 已正确把 `AbortError` / `TimeoutError` 归为 `retryable_network`（见 `src/lib/retry.ts:18-19`），无需改 retry.ts。

**大白话：** 打电话过去如果 30 秒还接不通，就挂掉重拨一次。拨完还不通，走失败流程，在 `markOcrFailure` 里明确写"超时了"，让用户看到"网络不通"而不是白屏 5 分钟。

- [ ] **Step 3：包裹 classify-pdf fetch（第一处 await fetch）**

把：

```ts
const classifyUrl = `${ocrBase}/classify-pdf`
const classifyRes = await fetch(classifyUrl, {
  method: 'POST',
  headers: await buildOcrHeaders(classifyUrl),
  body: JSON.stringify({ r2_object_key: r2ObjectKey, book_id: bookId }),
})
```

改成：

```ts
const classifyUrl = `${ocrBase}/classify-pdf`
const classifyRes = await retryWithBackoff(
  async () => fetch(classifyUrl, {
    method: 'POST',
    headers: await buildOcrHeaders(classifyUrl),
    body: JSON.stringify({ r2_object_key: r2ObjectKey, book_id: bookId }),
    signal: AbortSignal.timeout(30_000),
  }),
  { maxAttempts: 2, baseMs: 1000 }
)
```

注意：`buildOcrHeaders` 放在 retry 回调内部——重试前会重新取一次 header（防 token 过期边界 case）。

- [ ] **Step 5：给 ocr-pdf fetch（第三处 fire-and-forget）加 timeout——不包 retry**

`ocr-pdf` 是 fire-and-forget（`void fetch(...)`），包 retryWithBackoff 会让它改变行为（await 堵住主流程）。只加 timeout 即可，让失败能尽快进入 `.catch` 分支写 `markOcrFailure`。

把：

```ts

  method: 'POST',
  headers: ocrPdfHeaders,
  body: JSON.stringify({
    r2_object_key: r2ObjectKey,
    book_id: bookId,
    classifications: pages,
  }),
})
```

改成：

```ts
void fetch(ocrPdfUrl, {
  method: 'POST',
  headers: ocrPdfHeaders,
  body: JSON.stringify({
    r2_object_key: r2ObjectKey,
    book_id: bookId,
    classifications: pages,
  }),
  signal: AbortSignal.timeout(30_000),
})
```

- [ ] **Step 6：最外层 catch 的 error 信息保留原样**

`runClassifyAndExtract` 末尾的 `catch (error) { await markOcrFailure(bookId, `upload flow error: ${String(error)}`) }` 已经能捕获重试耗尽后的 `AbortError`，`String(error)` 会包含 "timeout" 或 "aborted" 字样，满足验收标准 (b) "lastError 字段明确（含 timeout 或 retry 信息）"。不需额外改动。

-- scripts/cleanup-stuck-books-m4.6.sql
-- M4.6 清理 OCR hang 期间 stuck 的 books（book id 10, 11, 12）
-- 流程：先跑 SELECT 段 → 把结果贴进 docs/journal/2026-04-22-stuck-books-snapshot.md
--      → 再跑 DELETE 段

-- STEP 1: SELECT - 验证 + 备份（把输出存到 journal snapshot）

  id,
  user_id,
  title,
  file_size,
  parse_status,
  kp_extraction_status,
  upload_status,
  ocr_total_pages,
  scanned_pages_count,
  file_path,
  created_at
FROM books
WHERE id IN (10, 11, 12)
ORDER BY id;

-- 同时看看 modules 表里挂了多少行（cascade 删之前先数一下）
SELECT book_id, COUNT(*) AS module_count
FROM modules
WHERE book_id IN (10, 11, 12)
GROUP BY book_id
ORDER BY book_id;

-- STEP 2: DELETE - 实际清理（跑 Step 1 确认无误后再跑这段）
-- ============================================================

-- 验证：应返回 0 行
SELECT id, title FROM books WHERE id IN (10, 11, 12);
