# M3.5 里程碑衔接修复 — 设计文档

> 修复 M3→M4 的 3 个接入断点，确保复习系统有数据基础。

---

## 背景

M3（考官 AI）完成后，代码审计发现 3 个问题阻碍 M4（复习系统）启动：
1. 测试通过不触发复习调度
2. clusters.next_review_date 与 review_schedule.due_date 语义冗余
3. reviewer prompt 模板是乱码 UTF-8

这 3 个问题与 `docs/architecture.md` 中的 3 个 ⚠️ 标记完全对应。

---

## T1: 测试通过 → 触发复习调度

### 现状

`src/app/api/modules/[moduleId]/test/submit/route.ts:235-237`

测试通过（≥80%）时只执行：
```ts
db.prepare('UPDATE modules SET learning_status = ? WHERE id = ?').run('completed', id)
```

不创建 review_schedule，不更新 cluster P 值。M4 复习流程无法启动。

### 改动

在 `db.transaction(...)` 回调体内、`if (isPassed)` 块中、`learning_status = 'completed'` 之后、闭合 `})` 之前追加：

**1. 创建 review_schedule 第一条记录**

```sql
INSERT INTO review_schedule (module_id, review_round, due_date, status)
VALUES (?, 1, date('now', '+3 days'), 'pending')
```

**2. 按 cluster 维度更新 P 值**

逻辑：遍历该模块的所有 cluster，统计每个 cluster 下测试题的正确率。

- **该 cluster 下所有题全对**：`consecutive_correct += 1`；若 `consecutive_correct >= 2`，则 `current_p_value = min(current_p_value + 1, 5)`
- **该 cluster 下有错题**：`consecutive_correct = 0`，`current_p_value = max(current_p_value - 1, 1)`

数据来源：
- test_questions 关联 kp_id → knowledge_points.cluster_id → 得到每道题的 cluster 归属
- test_responses 的 is_correct 字段 → 判断对错

### 边界情况

- 某个 cluster 在本次测试中没有被出题覆盖 → 不更新该 cluster（保持原值）
- P 值上限 5、下限 1
- review_schedule 只在首次通过时创建：`SELECT id FROM review_schedule WHERE module_id = ? AND review_round = 1`，有记录则跳过 INSERT，避免重测时重复创建

### 不做的事

- 不修改前端
- 不实现复习流程本身（M4 内容）

---

## T2: 删除 clusters.next_review_date

### 现状

`src/lib/db.ts:110` clusters 表有 `next_review_date TEXT` 字段，但：
- 无代码读取或写入此字段
- 复习调度已有 `review_schedule.due_date`（module 级），语义冗余
- `docs/architecture.md` 标记为 ⚠️

### 决策

**删除此字段。** 理由：M4 spec 按 module 级调度，无 cluster 级调度需求。未来需要时加回成本极低。

### 改动

1. 从 `db.ts` clusters 表 CREATE TABLE 语句中移除 `next_review_date TEXT` 行
2. 添加 migration 逻辑：`ALTER TABLE clusters DROP COLUMN next_review_date`（SQLite 3.35+ 支持）
3. 检查全代码库确认无引用（审计已确认无引用）

## T3: 重写 reviewer prompt 模板

### 现状

`src/lib/seed-templates.ts:340-342`

reviewer / review_generation 的 template_text 是损坏的 UTF-8 编码（乱码），无法被 AI 正确理解。对比：examiner 模板已在 M3 中用正常中文重写。

### 改动

参照 examiner 模板格式，用正常中文重写 reviewer 模板：

```
你是一位复习出题专家。

## 任务
根据聚类和 P 值出复习题。P 值越低，说明学生对该聚类掌握越差，需要出更多题。

## 复习规则
{review_rules}

## 聚类及 P 值
{clusters_with_p}

## 对应知识点
{kp_table}

## 历史错题
{past_mistakes}

## 出题策略
- P=1（薄弱）：出 3 题，优先覆盖历史错题对应的 KP
- P=2（一般）：出 2 题
- P≥3（掌握较好）：出 1 题
- 历史错题对应的 KP 必须优先覆盖
- 题型参照考试题型：单选题、计算题、思考题、C2 评估题
- 题目难度与原始测试持平，不刻意提高或降低

## 质量自检（内部执行，不输出）
- 所有 P≤2 的 cluster 都被覆盖
- 历史错题对应的 KP 都出了题
- 题目不与最近一轮复习重复（如有 {recent_questions}）

## 输出要求
返回 JSON:
{
  "questions": [
    {
      "cluster_id": 0,
      "kp_id": 0,
      "type": "multiple_choice|calculation|essay|c2_evaluation",
      "text": "题目文本",
      "options": ["A. ...", "B. ...", "C. ...", "D. ..."],
      "correct_answer": "正确答案",
      "explanation": "解析"
    }
  ]
}
```

### 模板变量

沿用现有占位符 + 新增 1 个：
- `{review_rules}` — 复习间隔规则（3/7/15/30/60 天）
- `{clusters_with_p}` — 聚类列表及当前 P 值
- `{kp_table}` — 知识点表
- `{past_mistakes}` — 历史错题记录
- `{recent_questions}`（新增）— 最近一轮复习的题目，用于去重。M4 实现调用方时填充；M3.5 只在模板中声明占位符，seed 时不需要提供值

### seed 更新策略

修改 `seed-templates.ts` 中的模板文本。对已有数据库，seed 逻辑在 `role='reviewer' AND stage='review_generation'` 记录已存在时需做 UPDATE（与 examiner M3 重写时相同策略）。
