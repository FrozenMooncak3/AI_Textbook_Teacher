## 步骤 1 · 小 PDF baseline（验证基础链路）

1. 进入"上传"页
2. 选小 PDF → 确认上传
3. confirm 成功后自动跳转到 preparing 页
4. 开始计时

**期望：** 60 秒内 preparing 页显示 `firstModuleReady=true`，可点"开始学习"进入模块地图。

**若失败：** 直接跳步骤 3 记录反馈。

---

## 步骤 2 · 14MB 真书（验证 hang 是否真修好）

1. 进入"上传"页
2. 选 14.2MB 教材 PDF → 确认上传
3. confirm 成功后自动跳转到 preparing 页
4. 开始计时

**期望：** 60 秒内 preparing 页显示 `firstModuleReady=true`。

**若通过：** 把"通过 + 时间戳 + 大致用时"截图或文字告诉 Claude，走 T6A 收尾。

**若失败：** 跳步骤 3 记录反馈。

---

## 步骤 3 · 失败时反馈清单

同时截图 / 复制以下信息发给 Claude（越全越能快速定位）：

1. **Preparing 页截图**
   - 含 `lastError` 字段内容
   - 含当前时间戳（浏览器右下角系统时钟）

2. **Vercel function logs 第 1 屏**
   - 打开 https://vercel.com/ → project `ai-textbook-teacher` → 最新 deployment → **Logs** tab
   - 搜索关键词：`runClassifyAndExtract` / `ocr-pdf` / `timeout` / `aborted` / `book_ocr_failed`
   - 截前 20-30 行

3. **具体卡了多久**
   - 从点"确认上传"到 `lastError` 出现，大致多少秒

4. **lastError 的内容**
   - 有 `timeout` / `aborted` 字样？
   - 有 HTTP 5xx？
   - 有 retry attempt 相关文字？

---

## 期望 Signal（Vercel logs）

### 成功路径

Vercel logs 应出现（至少一条）：

```
book_upload_classified bookId=<id>, text=<n>, scanned=<n>, mixed=<n>
```

或：

```
triggerReadyModulesExtraction bookId=<id>
```

### 失败路径（retry 耗尽）

Vercel logs 应出现：

```
book_ocr_failed bookId=<id>, upload flow error: ... TimeoutError / AbortError ...
```

或：

```
book_ocr_failed bookId=<id>, upload flow error: ... aborted ...
```

**关键：** `lastError` 字段要**明确说超时 / 被中止**，而不是之前那种 `TypeError: fetch failed` 白屏 4-6 分钟的状态。

---

## 验收标准（出自 spec §8 / plan T5）

- ✅ book 11（14MB）重传 → preparing 页 **60 秒内** `firstModuleReady=true`
- ✅ 失败 case：**30 秒内** `lastError` 字段明确（含 "timeout" / "aborted" / retry 信息）— 不是 5 分钟白屏
- ✅ Vercel function logs 含成功（classified）或失败（ocr_failed timeout）信号之一

---

## 不应再出现

- 4-6 分钟白屏、`lastError` 全程空的"卡死状态"
- Vercel logs 无任何 `runClassifyAndExtract` / `book_ocr_failed` 相关输出
- `TypeError: fetch failed` 无其他上下文

若仍出现上述，说明两拳没打中，走 T6B 进入诊断阶段（加 instrumentation）。
