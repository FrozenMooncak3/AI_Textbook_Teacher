# 调研 3：AI 教学对话 prompt 编码

**调研日期**: 2026-04-11
**用途**: 教学系统设计——如何把教学法规则编码成 AI prompt，让 AI "教得像好老师"
**前置**: 调研 1（知识类型分类）、调研 2（教学法匹配）

---

## 核心问题

把教学法写进 prompt，AI 就能教好吗？

研究结论：**是的，但有条件**。关键不是"告诉 AI 你是老师"，而是用结构化约束限制 AI 的行为空间——把"应该做什么"和"不能做什么"都编码进去。

---

## 现有系统的 prompt 编码方案

### 1. AutoTutor（学术研究，17 年验证）

**对话框架**：五步循环
1. Tutor 提出深层问题
2. 学生给出初始回答
3. Tutor 用 12 种对话动作之一回应（pump / hint / prompt / splice / elaborate / summarize + 5 级反馈）
4. 合作完善答案
5. 评估理解（后来取消——学生的"懂了"不可靠）

**编码关键**：
- **期望文本 (Expectation)**：每个教学点预设"理想回答"，用 LSA（潜在语义分析）比对学生回答与期望的匹配度
- **误解清单 (Misconception)**：预设常见错误，匹配到时触发专门纠正
- **对话动作选择器**：根据匹配度自动选择下一步动作——匹配高→pump/summarize，匹配低→hint/prompt

**对产品的启示**：
- "期望文本"思路可直接用——KP 提取时已经有知识点内容，这就是"理想回答"
- 12 种对话动作对 LLM 太细碎；LLM 的优势是自然语言理解，不需要预定义动作列表
- 五步框架适合保留但需要简化——3 步轻量版 + 5 步完整版（见下文推荐方案）

### 2. Khanmigo（商业产品，最广泛使用）

**prompt 核心**：
```
"You are a Socratic tutor. I am a student. 
Don't give me answers to my questions but lead me to get to them myself."
```

**设计特征**：
- 单一教学法（纯苏格拉底式），不根据知识类型切换
- 用自定义 prompt 防止直接给答案
- 绑定 Khan Academy 内容库作为知识源
- 限制回复长度（防止信息过载）

**效果数据**：
- 使用 Khanmigo 的学生比用通用 ChatGPT 的学生理解概念更好
- 用户从 2023-24 年 6.8 万增长到 2024-25 年 70 万+

**对产品的启示**：
- 苏格拉底法只适合部分知识类型（概念型、分析型），不适合事实型和过程型
- 但"绝不直接给答案"的 prompt 约束是所有类型都需要的（认知卸载防护）
- 回复长度限制是实用的 UX 约束

### 3. 认知学徒制 AI（arxiv 2601.19053，2025 研究）

**prompt 架构**：多层结构
- **指令层 (Instructions)**：核心教学行为和对话策略
- **知识文件层 (Knowledge files)**：反馈方法、示例对话模式、领域原则
- **对话启动器 (Conversation starters)**：单一入口
- **阶段逻辑 (Phase-based)**：内部分阶段，每阶段有明确目标

**七条设计准则**：
1. 结构化反馈循环：对话分三阶段——澄清目标 → 诊断讨论 → 反思探索
2. 先澄清目标：帮学生从模糊关切到具体问题
3. 嵌入专家推理：让 AI 的思维可见（Verbalize）、可泛化（Generalize）、有例子（Exemplify）
4. 渐进式指导序列：Coaching → Scaffolding → Modeling，不是直接给方案
5. 支持性环境：情感支持，营造心理安全
6. 视觉/上下文锚定：要求 AI 先确认理解再给建议
7. 显式结构：用标记、标题、里程碑提供清晰进度

**量化结果**：
| 指标 | 标准 ChatGPT | 认知学徒制 prompt |
|------|------------|----------------|
| 对话轮数 | 3.1 | 6.2（2 倍） |
| 学生追问数 | 0.7 | 2.4（3.4 倍） |
| 学生回答长度 | 67 词 | 156 词（2.3 倍） |
| 脚手架使用 | 0 次 | 26 次 |

### 4. 双层 prompt 架构（LPITutor，2025 研究）

**架构**：
- **静态层**：包含教学法模板（"用与 [学生背景] 相关的例子解释 [概念]"）
- **动态层**：注入 RAG 检索的课程内容 + 学习者元数据（进度、薄弱点）

**关键发现**：
- RAG 锚定课程材料可显著降低幻觉
- prompt 护栏维护学术诚信（不泄答案）
- 情感脚手架提升学习坚持度

## 推荐 prompt 编码方案

### 架构：三层 prompt

```
┌─────────────────────────────────────────────┐
│ 第一层：角色 + 教学法规则（静态，按知识类型切换）   │
│   - 教学人格                                  │
│   - 教学法核心流程                              │
│   - 禁止行为（认知卸载防护）                      │
│   - 质量控制规则                               │
├─────────────────────────────────────────────┤
│ 第二层：知识锚定（动态，从 DB 注入）               │
│   - KP 内容（原文出处）                         │
│   - KP 类型 + 教学法选择                        │
│   - 上下文（前后 KP 关系、模块摘要）              │
│   - 学生已有回答/历史表现                        │
├─────────────────────────────────────────────┤
│ 第三层：对话管理（动态，根据对话进展更新）           │
│   - 当前对话阶段（教学进度）                      │
│   - 学生回答质量评估                            │
│   - 下一步动作指令                              │
└─────────────────────────────────────────────┘
```

### 第一层：角色 + 教学法规则

**按知识类型切换 5 套 system prompt 模板**：

| 知识类型 | 教学人格 | 核心规则 |
|---------|---------|---------|
| 事实型 | 教授型（清晰高效） | "先给类比，再给正式定义，然后追问'为什么用这个词'"。不用苏格拉底法。 |
| 概念型 | 导师型（引导思考） | "先问学生已知什么，用类比建模，给正反例，追问'为什么'和'什么条件下'"。 |
| 过程型 | 教练型（耐心示范） | "完整演示第一遍，标注每步原因。第二遍留空让学生填。第三遍学生独立做。" |
| 分析型 | 师傅型（展示思维） | "先展示你的完整推理过程。然后给类似情境让学生做，只在关键点提示。" |
| 评估型 | 同行型（一起讨论） | "给案例，让学生先判断。然后提反面论点。最后改变条件问'结论变吗'。" |

### 第二层：知识锚定

```
你正在教的知识点：
- 名称：{kp_name}
- 类型：{kp_type}
- 原文内容：{kp_content}
- 出处：{source_text}（教材第 X 页）
- 前置知识点：{prerequisite_kps}

教学要求：
- 核心知识点必须有原文出处支撑
- 你可以用自己的语言重新组织，但不能编造教材中没有的事实
- 补充的例子或类比必须标注"这是补充说明"
```

### 第三层：对话管理

每轮对话后，系统评估学生回答并更新 prompt：

```
学生回答质量评估：
- 秒答正确 → confidence 高 → 加快节奏
- 犹豫但对 → confidence 中 → 正常节奏
- 答错 → confidence 低 → 深入教学/换方法
- 表达困惑 → 启动困惑诊断（见下文）
```

## 质量控制编码

### 认知卸载防护（最重要）

写进 prompt 的硬规则：

```
绝对禁止：
1. 不得在学生尝试回答之前给出答案
2. 不得在一次回复中给出完整解答（每次只推进一步）
3. 不得用"答案是 X"这种直接陈述——用引导问题代替
4. 当学生直接问"答案是什么"时，回应"让我们一步步来想"

替代方案：
- 学生不会 → 给提示，不给答案
- 学生部分对 → 追问"还有呢？"
- 学生完全偏 → 用反问暴露矛盾（"如果按你说的逻辑，那 X 情况怎么解释？"）
```

**数据支持**：Harvard RCT 显示，有 prompt 护栏的 AI 辅导组在练习题上得分比无护栏组高 127%，且在闭卷考试上不输对照组。无护栏组练习题得分高 48%，但闭卷考试下降 17%。

### 困惑诊断（投资 spec 可直接借鉴）

```
当学生表达困惑时，定位层级并选择对应策略：
1. 术语不懂 → 换一个日常类比重新解释
2. 概念关系不清 → 拆成更小的步骤，一步一步建立
3. 与已知知识冲突 → 找到冲突点，用反例暴露误解，再重建
4. 信息过载 → 简化到核心要点，其他暂时搁置

验证理解时不要问"懂了吗"（学生会说"懂了"但并没懂）。
让学生用自己的话复述——如果能说出来，才算理解。
```

### 反馈质量（投资 spec 可直接借鉴）

```
回应学生错误时：
- 不说"错了"——用间接暗示法（提问让学生自己发现矛盾）
- 错误分类处理：
  - 手误/笔误 → 快速指出，不深究
  - 概念性错误 → 三步处理：识别错误 → 追问理解原因 → 纠正
  - 系统性误解 → 停下来重新教学，换一种方式讲
```

### 回复长度控制

```
每次回复控制在 100-200 字以内。
教学时：一次只教一个要点，不要一口气讲三个。
反馈时：先说核心判断（对/部分对/偏了），再用 1-2 句解释。
绝不写超过 3 段的长回复。
```

## 对话流程编码：轻量版 vs 完整版

根据 KP 的重要性和复杂度，分两种对话流程：

### 轻量版（3 步，用于事实型 KP 和简单概念型 KP）

```
Step 1: 呈现 — AI 用类比+定义呈现知识点
Step 2: 确认 — 让学生复述或回答一个确认问题
Step 3: 收尾 — 根据回答判断：对→"很好，下一个"；错→简短纠正后继续
```

预计 2-3 轮对话，1-2 分钟。

### 完整版（5 步，用于概念型/过程型/分析型/评估型）

```
Step 1: 激活 — 问学生已经知道什么 / 提出深层问题
Step 2: 展开 — 按知识类型对应的教学法展开教学
Step 3: 练习 — 让学生在新情境中应用
Step 4: 反馈 — 根据回答给出有针对性的反馈
Step 5: 整合 — 让学生用自己的话总结学到了什么
```

预计 4-6 轮对话，3-5 分钟。

### 选择规则

| 条件 | 版本 |
|------|------|
| 事实型 KP | 轻量版 |
| 概念型 KP 且复杂度低 | 轻量版 |
| 概念型 KP 且复杂度高 | 完整版 |
| 过程型 / 分析型 / 评估型 | 完整版 |

## 与投资 spec prompt 设计的对比

| 维度 | 投资 spec | 通用教材方案 |
|------|----------|------------|
| prompt 模板数 | 1 套（投资 tutor） | 5 套（按知识类型切换） |
| 对话步数 | 固定 5 步 | 3 步轻量 / 5 步完整 |
| 知识锚定 | 读取 YAML 文件 | 从 DB 读取 KP + 原文 |
| 质量控制 | 10 维度硬编码 | 核心 4 条硬规则 + 类型专属规则 |
| 人格切换 | 4 种人格 | 5 种人格 |
| 困惑诊断 | 4 层诊断 | 直接复用 |
| 反馈质量 | 间接暗示法 + 错误分类 | 直接复用 |
