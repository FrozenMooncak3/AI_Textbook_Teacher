## 1. 问题陈述

OCR 成本通过 D0 决策（仅接 text PDF + TXT，≤10MB / ≤100 页）已归零，**KP 提取 LLM 调用是 MVP 唯一的可变成本**。一本 369 页中文教材的全量提取已经把 Gemini 2.5 Pro 预付额度烧光（~8-15 元/本）。本文调研 8 个候选 LLM 在中文教材 KP 提取场景的横评，目标是把 single-book KP cost 压到 **≤1 元/本**，同时保住三件命根：(a) 中文章节理解质量、(b) JSON 严格输出（Stage 0 / 1 / 2 三阶段都要求 fenced json），(c) 中国大陆可达。`src/lib/services/kp-extraction-service.ts:39-48` 通过 `getModel()` 单一来源切换 provider，模型替换=改 env，所以这是个**易反悔**决策。

## 4. 场景化成本估算

**假设**：
- 一本教材 100 页 × 800 token/页 = 80K input token
- Stage 0（结构扫描）= 80K input + 4K output（一次）
- Stage 1（按 section 抽取）= 假 6 sections × (15K input + 8K output) = 90K in / 48K out
- Stage 2（quality check）= 30K in + 16K out
- 总计：input ~200K + output ~68K（一次完整提取）
- 1 USD = 7.2 RMB（2026-04-25 大致汇率）

## 5. 推荐方案

**Triage 三档：**

### 🥇 推荐 Plan A — DeepSeek V3.2（cheapest viable）
- **理由**：0.93 元/本达标；国内可达；现成 prompt 几乎不改（兼容 OpenAI 风格 response_format）；社区/工具生态成熟（vercel ai sdk 有 deepseek provider）
- **一次性风险**：JSON 偶发空响应（官方已认）→ 需要在 `parseJSON` 里加更激进的重试 + 切回兜底模型；2026-07-24 deepseek-chat 退役需准备 V4 切换
- **成本预期**：100 用户/晚 ≤ 100 元，可承受

### 🥈 推荐 Plan B — Qwen3-Max（国内站）（safest viable）
- **理由**：0.50 元/本；阿里云 SLA 比 DeepSeek 好；1M 上下文（vs DeepSeek 64K）；JSON 通过 prompt 注入稳定；不存在 deprecation 倒计时
- **一次性风险**：国际站价（$0.78/$3.90）比国内站贵 4-5 倍，海外用户路由要规划；prompt 兼容度需小调（Qwen 系建议 Hermes-style tool 描述）
- **成本预期**：100 用户/晚 ≤ 50 元，更安全

### 🥉 推荐 Plan C — Gemini 2.5 Flash（strategic hedge）
- **理由**：保持现有 SDK / prompt 不动，只换模型 ID；同一家厂商，迁移成本=0；JSON schema 模式生态最完善
- **一次性风险**：1.66 元/本仍不达标（≤1 元目标超 66%）；国内可达性问题不变
- **定位**：如果不想动技术栈、愿意高一档付费换稳定，作为暂时方案

### ❌ 排除候选
- **DeepSeek R1**：贵 3 倍买不到 3 倍质量提升，KP 抽取非推理任务
- **Yi-Lightning**：16K 上下文是结构性短板，textbook chunk 跑不动
- **Qwen-Turbo**：太小，KP 漏抽风险高（需先用一本书做 A/B 验证再考虑）
- **GLM-5**：能力档次确实更高（AA 50 vs Gemini Pro 35），但 1.94 元/本超目标，不如直接用 Pro

## 6. JSON 可靠度专项

`src/lib/services/kp-extraction-service.ts:94-115` 的 `parseJSON` 已经有两层兜底（fenced match + 引号修复），但 Stage 1 失败会丢失一个 section 的 KP 数据。各候选风险评估：

| 模型 | JSON 风险等级 | 失败模式 | 缓解措施 |
|------|--------------|---------|---------|
| Gemini 2.5 Pro/Flash | 低 | schema 模式键序被强制字母序（CoT 失效，但当前 prompt 不依赖 schema） | 维持 fenced markdown 模式不变 |
| DeepSeek V3.2 | **中-高** | 官方承认"may occasionally return empty content" | 加 retry + 切到 Qwen-Plus 兜底 + Sentry 报警 |
| DeepSeek R1 | 中 | CoT 输出可能漏 fenced 标记 | parseJSON 已能处理 |
| Qwen3-Max / Plus | 中 | 不支持 native json_schema，需 prompt 注入示例 | 现 prompt 已含 example，无需改 |
| Qwen-Turbo | 中-高 | 小模型偶发字段漂移 | 不推荐用于 Stage 1 |
| GLM-4.7 / 5 | 中 | function call native，json_object 模式文档稀少 | 需先做 50 chunk 抽样验证 |
| Yi-Lightning | (未查到) | 16K 上下文先把 chunk 切碎 | 排除候选 |

> Quote: "DeepSeek's JSON Output feature has a significant limitation: 'the API may occasionally return empty content.' The documentation indicates they are 'actively working on optimizing this issue'" [https://api-docs.deepseek.com/guides/json_mode] [S]
