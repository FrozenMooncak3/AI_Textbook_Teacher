# AI 成本趋势与护城河分析

**调研日期**: 2026-04-11
**用途**: 商业策略——定价模型、成本预算、产品防御性设计

---

## AI Token 成本趋势

### 历史趋势

| 时间点 | GPT-4 等效性能成本 (per M tokens) | 倍数变化 |
|--------|----------------------------------|---------|
| 2022 年末 | $20 | 基准 |
| 2024 年中 | $2-5 | 4-10x 下降 |
| 2025 年中 | $0.40-1.00 | 20-50x 下降 |
| 2026 年 | $0.15-0.40 | 50-130x 下降 |

### 具体价格变化

- GPT-4o mini (2024-07): $0.15/$0.60 per M tokens — 比 GPT-3.5 Turbo 便宜 60%
- GPT-5 nano (2025): $0.05/$0.40 per M tokens
- DeepSeek R1 (2025): $0.55/$2.19 per M tokens — 比主流低约 90%
- 价格下降速率：每年 9x-900x 不等（取决于任务类型），中位数约 40x/年

### 下降驱动因素

1. 模型架构效率提升（MoE、稀疏注意力）
2. 硬件成本下降（GPU 单位算力成本持续降低）
3. 激烈竞争（DeepSeek、Mistral 等开源模型压低价格）
4. 推理优化技术（量化、蒸馏、投机解码）

### 对产品的影响

**当前成本估算（Claude Sonnet）**：
- 每个 KP 教学对话（5-10 轮）: ~3K-8K tokens → ~$0.02-0.06
- 一个模块（10 KP）完整教学: ~$0.2-0.6
- 一本书（10 模块）: ~$2-6
- 每用户每月（假设学 2 本书）: ~$4-12

**1 年后预估**：
- 成本可能下降 5-10x → 每用户每月 $0.5-2.5
- 复杂多轮教学对话在成本上完全可行

**结论**: Token 成本不是瓶颈。现在开始设计"正确但 token 密集"的教学方案是合理的。

## 护城河分析

### 什么不是护城河

| 因素 | 为什么不是护城河 |
|------|-----------------|
| "我用了 AI" | 所有人都能用，API 接入零门槛 |
| "我有漂亮 UI" | 前端可以复制 |
| "我能分析 PDF" | NotebookLM 免费做，Google Docs 也能做 |
| "我有内容库" | 任何有资金的团队可以雇专家快速构建 |
| "我价格便宜" | Token 成本全行业下降，价格战无法持续 |

### 什么是护城河

**1. 教学方法论的产品化（最核心）**

把"不同知识类型用不同方法教"做成自动化体验。这需要：
- 教育学研究 → 知识分类 → 教学法匹配 → prompt 编码 → 效果验证 → 迭代优化

每一次迭代都提升教学质量。这个积累过程竞争对手需要同样长的时间走一遍。

**为什么大公司不容易追**: 大公司追求通用性（一个产品服务所有场景），在单一场景的深度上很难投入足够。Google 可能把 NotebookLM 做成"什么都能做"，但在"用正确方法教学"这个维度上做不深。

**2. 学习数据飞轮**

```
用户学习数据（什么类型知识容易卡住、什么教学策略效果好）
    → 改进教学策略 → 更好的效果
        → 更多用户 → 更多数据 → 更精准的改进
```

需要先有用户量才能启动。

**3. 垂直场景深度**

专注"教材深度学习"这个场景，做到 10x 好于通用 AI。通用 AI 在这个场景上永远是"够用但不专精"。

**4. 用户习惯和数据锁定**

用户的学习进度、知识追踪、复习计划都在你的系统里。迁移成本 = 重新学一遍。

### 护城河时间窗口

**紧迫性**: 在大公司把教学功能做好之前，先建立用户基础和数据飞轮。

**预估窗口**: 
- NotebookLM 当前没有真正的教学法。但 Google 可以在 6-12 个月内加入。
- 你的窗口期大约是 12-18 个月——在这期间建立足够深的教学方法论积累和用户基础。

### 一夜间被替代的风险

**结论: 不太可能一夜间，但长期有风险。**

- 教学方法论不是 AI 模型自动学会的。GPT-10 不会自动知道"定义类知识用直接讲解+间隔重复最有效"。
- 但如果 Google 投入教育学团队做这件事 + 免费 + 用户基础，威胁很大。
- 你的防守策略：速度 + 深度 + 垂直场景。
