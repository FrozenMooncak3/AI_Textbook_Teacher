# 调研 1：通用知识类型分类方案

**调研日期**: 2026-04-11
**用途**: 教学系统设计——确定 KP 分类体系，驱动教学法匹配

---

## 研究的三个主流分类框架

### 1. Anderson-Krathwohl 修订版 Bloom 分类法（2001）

两个维度的交叉矩阵：

**知识维度**（内容是什么）：
| 类型 | 定义 | 教材中的例子 |
|------|------|-------------|
| **事实性知识 (Factual)** | 独立的信息片段：术语、细节、数据 | "GDP = 国内生产总值"、"水的沸点是 100°C" |
| **概念性知识 (Conceptual)** | 要素之间的关系：分类、原理、理论、模型 | "供需关系如何影响价格"、"自然选择理论" |
| **过程性知识 (Procedural)** | 做事的步骤：算法、方法、技术 | "如何做 t 检验"、"求解一元二次方程" |
| **元认知知识 (Metacognitive)** | 关于认知的知识：策略意识、自我调节 | "我应该用什么方法学这类问题"（不适合 AI 直接教） |

**认知过程维度**（要求学生做什么）：
```
记忆 (Remember) → 理解 (Understand) → 应用 (Apply) → 分析 (Analyze) → 评价 (Evaluate) → 创造 (Create)
```

**优点**: 两维交叉覆盖全面；学术引用最多；被教育界广泛使用。
**缺点**: 元认知维度在 AI 教学场景不易操作；6 个认知层级对 AI prompt 编码太细。

### 2. Merrill 组件展示理论 (CDT)

**内容类型**（4 类）：
| 类型 | 定义 | 对应的教学形式 |
|------|------|--------------|
| **事实 (Facts)** | 信息片段：数字、名称、事件 | 呈现规则 + 举例 + 记忆练习 |
| **概念 (Concepts)** | 共享特征的分类体系 | 定义 + 正反例 + 分类练习 |
| **过程 (Procedures)** | 解决问题的步骤序列 | 步骤演示 + 部分提示练习 + 独立练习 |
| **原理 (Principles)** | 解释"为什么"的因果规则 | 因果链展示 + 预测练习 + 变式应用 |

**表现层级**（3 级）：记忆 → 使用 → 发现/泛化

**优点**: 分类直接对应教学策略，实操性强；"原理"类别独立出来比 Bloom 更实用。
**缺点**: 不覆盖高阶分析/评价能力；为传统教学设计，不针对 AI 场景。

### 3. 当前 App 的 4 类分类

| 类型 | 认知层级 | 教材例子 |
|------|---------|---------|
| 定义类 | 记忆/理解 | "什么是 GDP" |
| 计算类 | 应用 | "计算资产负债率" |
| C1 判断类 | 分析 | "给定数据判断假设是否成立" |
| C2 评估类 | 评价 | "评价两种理论的优劣" |

**优点**: 简洁，已在 KP 提取中验证可行。
**缺点**: "定义类"太宽泛——"GDP 的全称"和"供需关系如何影响价格"都叫定义类，但教法完全不同。

---

## 推荐方案：5 类通用知识类型

基于三个框架的优缺点和产品需求，推荐将"定义类"拆分为事实型和概念型，形成 5 类分类：

| 类型 | 对应 Bloom 层级 | 定义 | 教材中的典型表现 |
|------|----------------|------|-----------------|
| **事实型 (factual)** | 记忆 | 独立的信息片段：术语、定义、数据、日期、人名 | "GDP = 国内生产总值"、"细胞分裂有丝分裂和减数分裂两种" |
| **概念型 (conceptual)** | 理解 | 要素间的关系：原理、分类体系、理论、因果链 | "供需关系如何影响价格"、"自然选择的三个条件" |
| **过程型 (procedural)** | 应用 | 做事的步骤：公式计算、算法、方法、流程 | "如何做 t 检验"、"计算资产负债率"、"求解二次方程" |
| **分析型 (analytical)** | 分析 | 基于规则或模式的判断和推理 | "给定数据判断假设是否成立"、"判断这个论证是否有效" |
| **评估型 (evaluative)** | 评价 | 多维权衡、论证评价、无标准答案的判断 | "比较凯恩斯和弗里德曼的通胀理论"、"评价这个实验设计的优劣" |

### 为什么是 5 类，不是 4 类或 6 类

**为什么拆分"定义类"→ 事实型 + 概念型**：
- 事实型（"什么是 GDP"）需要直接讲解 + 间隔重复，认知要求低
- 概念型（"供需如何影响价格"）需要类比建直觉 + 追问"为什么"，认知要求高
- 两者的最优教学法截然不同：直接记忆 vs 深度理解
- 投资 spec 的"定义类"已经暗含"理解性定义追问"规则，说明 4 类不够精细

**为什么不加"创造型"（Bloom 第 6 级）**：
- 教材学习场景中极少要求学生"创造"新知识
- "创造"更适合项目式学习、毕业设计等场景，不在 MVP 范围内

**为什么不加"元认知型"（Anderson-Krathwohl 第 4 维度）**：
- 元认知（"我应该怎么学这个"）不是教材内容，是学习策略
- 产品通过系统设计实现元认知（进度追踪、知识诊断），不需要作为 KP 类型

### 与现有 4 类的映射

| 现有分类 | 新分类 | 迁移规则 |
|---------|--------|---------|
| 定义类 | 事实型 **或** 概念型 | KP 提取器判断：若只需记忆（术语、数据）→ 事实型；若涉及关系/原理/因果 → 概念型 |
| 计算类 | 过程型 | 直接映射 |
| C1 判断类 | 分析型 | 直接映射 |
| C2 评估类 | 评估型 | 直接映射 |

**迁移成本**：
- 提取器 prompt 需要更新：将"定义类"判断规则拆分为事实型/概念型的二级判断
- 已有 KP 数据的 type 字段需要迁移：定义类 → 由 AI 重新判断归类
- 其他 3 类直接重命名，无逻辑变化

### 识别规则（给 KP 提取器的判断标准）

| 类型 | 核心判断标准 |
|------|-------------|
| 事实型 | 能用一句话说清楚，不需要解释"为什么"。学生背下来就能用。 |
| 概念型 | 涉及"为什么"、"如何运作"、"什么条件下"。学生需要理解关系，不是背诵。 |
| 过程型 | 有明确步骤序列。学生需要"按步骤做"才算掌握。 |
| 分析型 | 给定信息后需要做判断/推理。有明确的判断标准或规则。 |
| 评估型 | 多个角度都有道理，需要权衡利弊。没有唯一正确答案。 |
