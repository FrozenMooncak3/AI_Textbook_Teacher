# MVP 范围扩展：扫描版 PDF + 多书种类

## 用户决策

两个功能从"未来做"升级为 **MVP 必须具备**，缺任何一个不上线：

1. **扫描版 PDF 支持** — 必须作为主要功能，不是附加。目前 OCR 管道只支持文字型 PDF，扫描版需要 PaddleOCR 全页 OCR
2. **多书种类** — 至少支持"教案课件型"（PPT/教案风格的 PDF）。目前系统只针对教材型文本优化

## 影响评估

### 扫描版 PDF
- 当前状态：PaddleOCR 已部署，`ocr_server.py` 有 `/ocr` 和 `/ocr-pdf` 端点
- 缺什么：扫描版 PDF 的 OCR 质量验证、可能需要图片预处理（去噪/倾斜校正）、大扫描 PDF 的性能优化
- 原停车位：T2（独立里程碑评估）→ 现在升级为 MVP 必做

### 多书种类（教案课件型）
- 当前状态：KP 提取的 prompt 模板只针对"教材型文本"设计（知识点提取、概念关系）
- 缺什么：教案/课件型 PDF 的提取策略（可能以"主题/要点"而非"知识点"为单位）、不同的 prompt 模板、可能需要 PPT→PDF 转换
- 原停车位：T2（独立里程碑）→ 现在升级为 MVP 必做

## 下一步

需要评估这两个需求的具体实现方案和工作量，纳入下一个里程碑规划。
