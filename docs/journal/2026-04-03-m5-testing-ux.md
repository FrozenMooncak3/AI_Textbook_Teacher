# M5 测试发现的 UX 问题：全局缺导航

**日期**：2026-04-03
**状态**：parked
**类型**：ux
**Tier**：T1（M5.5 最高优先级）

---

## 问题

**所有页面都没有返回键**，用户只能靠浏览器后退，体验很差。涉及：
- 上传页
- PDF 阅读器
- 模块学习页
- QA / 测试 / 复习页面
- 仪表盘
- 错题页
- 模块地图

另外仪表盘缺"阅读原文"直达入口。

## 更多 UX 问题（测试中发现）

3. **截图问 AI 没有 ESC 取消按钮**：打开后无法关闭（除了刷新页面）
4. **截图问 AI 应显示原图 + OCR 文字**：用户需要看到截图内容 + 可复制的 OCR 文字
5. **PDF 上传进度显示 (0/0)**：后端有真实进度（ocr_current_page/ocr_total_pages），前端未正确轮询显示
6. **PDF 阅读器功能太弱**：点"适应宽度"后无法恢复；缺少缩放控制、搜索、书签等基本功能。应替换为成熟 PDF 阅读器库（如 react-pdf-viewer）
7. **[BUG] OCR 完成后未自动触发 KP 提取**：parse_status=done 但 kp_extraction_status 停在 pending，用户只能手动调 /api/books/{id}/extract
8. **[限制] 大 PDF（230K+ 字符）KP 提取失败**：AI 单次处理不了整本书，需要文本分块 + 分段提取 + 合并

## 解决方向

M5.5 的导航 sidebar 应一次性解决导航问题：
- 全局侧边栏提供固定导航（首页、当前书、仪表盘、阅读器）
- 每个页面顶部有面包屑或返回链接
- 不再依赖浏览器后退

截图问 AI 改进归入 M5.5 UX 打磨。PDF 上传进度显示是 bug，需要修。
