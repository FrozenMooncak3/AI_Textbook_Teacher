# MVP 方向 Brainstorming

**类型**: decision
**状态**: open (设计中，未写 spec)
**日期**: 2026-04-06

## 背景

M0-M5.5 全部完成，核心学习流已跑通（上传 → KP 提取 → 阅读 → Q&A → 测试 → 复习）。用户开始思考 MVP 的目标人群和产品方向。

## 关键决策

### 1. MVP 目的

**验证假设**："留学生愿不愿意用 AI 学习系统学教材？"

不是做一个完美产品，是用最小代价验证需求。答案是 yes → 迭代；no → 趁早转向。

### 2. 目标用户

**不是**"不自律的学生"——不自律的人不会用任何学习工具。

**而是**"努力学了但方法低效的留学生"——上课了、课件看了，但成绩不匹配。他们的问题是学习方法错误（反复看笔记、画荧光笔、临考突击），不是缺乏意志力。

产品价值：自动执行被学习科学证明有效的方法（主动回忆、间隔重复、即时反馈）。

### 3. 竞品分析

发现一个已有留学生笔记软件（抖音 6w 赞、1.2w 粉），功能包括：
- 智能大纲（自动生成、跳转、书签）
- 翻译（整页翻译、划词释义、单词本）
- 文件夹与学习进度管理（打卡式追踪）
- AI 解题（圈题提交、提示式引导、手写交互）
- 错题本（自动收录、溯源跳转）

**核心区别**：
- 竞品 = **阅读辅助工具**（帮你"读懂"）
- 我们 = **学习系统**（确保你"学会"）

**战略判断**：不抄竞品功能（翻译、划词等），双方是互补不是对抗。我们的护城河是学习引擎（出题→测试→复习→错题追踪），竞品完全没有这些。

### 4. 推广策略

- 渠道：抖音、小红书
- 内容形式：录屏演示产品
- 用户来源：陌生人（不是朋友），所以产品不能有硬伤
- 定位话术："你不是不努力，你只是需要更聪明的学习方法"

### 5. MVP 范围

**MVP 做**（M6 里程碑）：
1. 修 2 个 data-state bug（test_ch1_2 报错 + 读财报白屏）
2. 大 PDF 分块提取（真实教材几百页，现在提取失败）
3. PDF 阅读器替换为成熟库（缩放/搜索/书签）
4. 用户账号系统（上线给陌生人用的前提）
5. 部署上线（公开可访问 URL）

**MVP 不做**（后续迭代）：
- 课件模式（PPT/短 PDF）→ MVP 后第一个迭代
- QA/复习时看原文
- 预生成系统
- 选文字问 AI
- 笔记+QA 联动
- 日历/思维导图/Hint
- 翻译功能（竞品的领域，不追）
- Subscription/付费

### 6. 课件 vs 教材的教学差异（未来参考）

| 维度 | 教材模式 | 课件模式 |
|------|---------|---------|
| 内容 | 完整文本，有解释 | 提纲，bullet point |
| 学生状态 | 从零学起 | 已上过课，复习查漏 |
| AI 角色 | 从原文提取、围绕原文出题 | 补全课件没说的、超出内容出题 |
| 阅读阶段 | 必须读完 | 可快速浏览或跳过 |
| 核心价值 | 把厚书读薄（压缩） | 把薄课件读厚（展开） |
