# MVP 必做：AI 再教一遍环节

## 用户决策

当前学习流程：模块拆解 → 读前指引 → **阅读原文** → Q&A → 测试 → 复习

缺失环节：阅读原文之后、Q&A 之前，缺少一个 **AI 根据书本内容主动教学** 的阶段。用户不应该只是自己读完就做题，AI 应该先把内容教一遍。

## 参考

用户指示参考"月饼投资项目的调研"中的教学环节设计。

## 影响评估

- 这会在学习状态流中新增一个阶段：`reading → **teaching** → qa`
- 需要新的 AI 角色或扩展教练角色（从"出题+反馈"扩展为"讲解+出题+反馈"）
- 需要新的 prompt 模板（基于 KP 内容生成讲解）
- 前端需要新的教学界面（可能类似笔记展示，但是 AI 主动讲解而非用户记笔记）
- 涉及 API、DB（可能需要存储教学内容）、前端三层变更

## 状态

MVP 必做，待详细方案设计。
