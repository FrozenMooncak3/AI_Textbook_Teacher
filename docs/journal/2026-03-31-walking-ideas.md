**类型**: idea
**状态**: parked

## OCR 后生成思维导图

用户提出：KP 提取或笔记生成后，自动生成思维导图展示知识结构。

**评估**：体验增强功能，非核心学习流程。需要前端思维导图渲染库。

**处置**：停车场，M5+ 体验打磨阶段评估。

## 笔记跳转原文

用户提出：笔记中的内容可以点击跳转到 PDF 原文对应位置。

**评估**：好功能，但依赖 PDF 页码/位置定位能力，技术复杂度中等。

**处置**：停车场，M5+ 评估。

## 右键选中多功能

用户提出：阅读 PDF 时，用鼠标选中某句话，右键弹出菜单：做笔记、字体颜色高亮等。

**评估**：富交互功能，需要前端大量工作（文本选择 API + 浮动菜单 + 标注持久化）。

**处置**：停车场，需独立评估工作量。

## 支持 PPT 和扫描版 PDF

用户提出：扩展输入格式，支持 PPT 和扫描版 PDF（非文字版）。

**评估**：扫描版 PDF 需要 OCR 能力（当前只支持文字版）。PPT 需要新的解析管线。影响面大，可能需要独立里程碑。

**处置**：停车场，MVP 后独立里程碑评估。

## 先面向扫描版 PDF 用户群体

用户提出：也许应该优先服务使用扫描版 PDF 的人群（教材多为扫描版），笔记功能对这个群体价值更大。

**评估**：用户定位和商业策略问题。扫描版 PDF 确实是教材场景的主力，但需要 OCR 基础能力先到位。

**处置**：停车场，与"支持扫描版 PDF"一起评估。
